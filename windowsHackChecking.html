Windows Hack Checking Workflow

A step-by-step methodology for exposing modifications and bypasses on Windows systems.

Phase 0: Rules & SS Timeline

SS Timeline

  • Wait for them to finish their fight, then /freeze.
  • Ask if they want to admit for a shorter ban. If yes, ban and screenshot.
  • If no, send an AnyDesk link via Discord/Telegram.
  • Once connected, SS them. Unfreeze if clean, ban if cheating.

Screensharer Rules

  • Do not browse personal files without initial suspicion.
  • Do not ban for simply having a cheat on their PC unless it was active/executed.
  • Do not download malicious files such as trojans.
  • Respect the person being SS'd and explain what each tool does.
  • Do not ban for jokingly admitting.

General Rules

  • Clearing the recycle bin after launch is bannable.
  • Using illegal modifications including macros is bannable.
  • A PC so degraded it cannot be SS'd is bannable.
  • Having critical Windows services turned off is bannable.
  • Unplugging a USB or having a FAT32 drive is bannable.
  • Suspect must not touch mouse/keyboard unless asked.

Important: ALWAYS record the entire screensharing session. Ensure Windows Defender and all third-party Antivirus software are disabled before running tools. All commands are run in CMD or CMD as Administrator — never in PowerShell directly.

Phase 1: Service Checker

Run this first. It verifies that all critical logging and system services are running, checks event log and prefetch state, and reports the recycle bin. Any disabled service from the list below is an immediate ban.

powershell Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass && powershell Invoke-Expression (Invoke-RestMethod https://raw.githubusercontent.com/praiselily/lilith-ps/refs/heads/main/Services.ps1)

Critical Services — Disabled = Immediate Ban

If any of these are stopped or disabled, the user has tampered with Windows to prevent forensic logging. There is no legitimate reason for any of these to be off on a gaming PC.

Additional Flags Reported by the Script

Beyond the services above, the checker also reports these states. Prefetch Disabled is an automatic ban. The others are contextual flags that need additional investigation:

  • Prefetch Disabled: Automatic ban. If HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Memory Management\PrefetchParameters\EnablePrefetcher is set to 0, the user manually disabled Prefetch to suppress .pf file creation. No legitimate reason exists.
  • PowerShell Logging Enabled: If Script Block Logging is enabled, it means someone turned it on — potentially to log their own commands for review, or a prior script enabled it. Check the event logs for Event IDs 4103/4104 to see what PowerShell commands were run.
  • Activities Cache Enabled: Windows Timeline is active, meaning app usage history may be stored in %LocalAppData%\ConnectedDevicesPlatform\. Worth cross-referencing with BAM.
  • USN Journal Cleared (Event ID 3079): Ban. Running fsutil usn deletejournal generates this event. The Journal is the primary record of all filesystem changes — deliberately clearing it is direct evidence tampering.
  • Event Logs Cleared (Event ID 1102): Ban. Clearing the Security event log produces Event ID 1102. This removes audit records of logins, process creation, and file access.
  • Recycle Bin recently modified: If the $Recycle.Bin modification timestamp aligns with the freeze time, the user likely just emptied it. Immediate ban.
Thread Suspension Note: A service can appear "Running" while its internal threads are suspended, silently disabling its logging function. This passes the service check. Confirm with System Informer in Phase 9.

Phase 2: Mod Analyzer

Run the Mod Analyzer before downloading the full tool suite. This script reads the running Minecraft process to inspect loaded JVM arguments, identifies disallowed or suspicious mods in the active classpath, and flags anomalies like unknown -javaagent entries.

MeowModAnalyzer (Tonynoh)

powershell -ExecutionPolicy Bypass -Command "Invoke-Expression (Invoke-RestMethod 'https://raw.githubusercontent.com/MeowTonynoh/MeowModAnalyzer/main/MeowModAnalyzer.ps1')"

YarpModAnalyzer (YarpLetapStan)

powershell -ExecutionPolicy Bypass -Command "Invoke-Expression (Invoke-RestMethod https://raw.githubusercontent.com/YarpLetapStan/PowershellScripts/refs/heads/main/YarpsModAnalyzer6.0.ps1)"

What to look for:

  • Disallowed mods flagged: Any mod explicitly identified as a banned cheat client.
  • Mods loaded from unexpected paths: JARs loading from %TEMP%, Downloads, or hidden directories rather than .minecraft\mods.
  • Unknown -javaagent flag: -javaagent:/path/to/file.jar pointing to an unrecognized file is a primary ghost client injection technique — it loads arbitrary code at JVM startup before anti-cheat initializes.
  • Classpath anomalies: Entries in the JVM classpath that don't correspond to any mod in the mods folder.

Phase 3: Tool Collection

Download the full suite of forensic tools using the SSTools collector. This populates C:\SSTools with everything needed for the remaining phases.

powershell -ExecutionPolicy Bypass -Command "Invoke-Expression (Invoke-WebRequest -Uri 'https://raw.githubusercontent.com/Lafferrr/SSTools/main/SSTools.ps1' -UseBasicParsing).Content"

Tools are placed in C:\SSTools. This includes JarParser, BAMParser, MacroScanner, Strings, JournalTrace, System Informer, and supporting utilities.

Phase 4: JarParser Analysis

JarParser scans Prefetch records and DcomLaunch memory strings to find all .jar files that have been executed on the system since logon. Crucially, it ignores the mods folder — its output reflects JARs that were actually run by the JVM externally, not the contents of any folder.

Interpreting the output (see Image 1 reference):

  • The tool reports its logon time cutoff — only results after that time are shown.
  • It checks DcomLaunch for strings containing -jar. If no strings are found there, it moves on.
  • It then runs a USN Journal analysis for JAVAW/JAVA Prefetch file deletions.
  • The key output line is "Total JAVA/JAVAW .pf delete/rename events after logon time." Any number above 0 means Prefetch files for Java were tampered with after login.

What is a Flag:

  • A .pf delete/rename event is reported: The user deleted or renamed a Prefetch file for JAVAW.EXE after logging in — direct evidence of evidence destruction. Immediate ban.
  • A known cheat JAR appears in the output: Any JAR name that is a recognized cheat client (e.g., VapeV4.jar, MeteorClient.jar) being listed confirms it was externally executed. Not every JAR listed is a cheat, but any that is a recognized client name is grounds for a ban.
Note: JARParser output showing a JAR was executed does not mean the file still exists. The trace in Prefetch/DcomLaunch persists after deletion.

Phase 5: USN Journal Trace

The USN (Update Sequence Number) Journal records every filesystem change on NTFS — creations, deletions, renames, data overwrites. It is difficult to fully clear without leaving its own trace. Focus on the timeframe around the start of the screenshare.

What to do:

Run JournalTrace. Filter for extensions: .jar, .exe, .dll, .pf, .lnk. Focus on the timeframe around when the SS started.

What is Normal:

Game logs updating (latest.log), OS-native temp file creation/deletion, launcher cache writes.

What is a Flag:

  • FILE_DELETE on a suspicious file right after the freeze — clicker.exe, VapeV4.jar, etc.
  • BASIC_INFO_CHANGE on C:\Windows\Prefetch\* files: The user set Prefetch files to Read-Only to freeze their timestamps and suppress new logging entries.
  • RENAME_OLD_NAME / RENAME_NEW_NAME pair at the same timestamp: A file was renamed to disguise it — e.g., cheat.exe → svchost.exe. Or two files swapped locations to make a cheat appear legitimate.
  • STREAM_CHANGE on a benign file: May indicate an Alternate Data Stream (ADS) being created to hide a payload inside an innocent host file.
  • Journal cleared (Event ID 3079): Ban. Running fsutil usn deletejournal is deliberate evidence destruction.

Phase 6: BAM Parser

BAM Parser reads the Background Activity Moderator registry key (HKLM\SYSTEM\CurrentControlSet\Services\bam\State\UserSettings\{SID}) and produces a clean report of every executable path that was run, when it was last run, and whether it ran in the current logon session.

The parser corrects device paths (e.g., \Device\HarddiskVolume3\) to readable drive letters, performs digital signature checks on each file, flags deleted files, and detects known cheat signatures (e.g., Slinky, Vape). It also cross-references the USN Journal for any file replacements.

What to look at:

  • Flagged files: The parser directly highlights files matching known cheat client signatures. These are your primary hits.
  • Not Signed / Unsigned: Any executable that is not digitally signed and was run in the current session is suspicious. Legitimate software is almost always signed.
  • In Instance (ran in current logon): Focus on these — these are files executed since the user last logged in, meaning during their active Minecraft session.
  • Common BAM flags to look for: Custom unsigned .exe files, Prestige Client injector entries, 198Macros, or any path pointing to Downloads, %TEMP%, or a hidden directory.
  • Deleted: A file that ran but no longer exists on disk — the user executed and deleted it. Strong evidence of cheating if the path or name is suspicious.
Also check: BAMReveal (Orbdiff) provides an additional view into BAM data with extended analysis capabilities.

Phase 7: MacroScanner & Imports Checker

MacroScanner

MacroScanner is a dedicated macro detection tool. It performs several checks automatically:

  • Script file scanning: Scans the system for script files with extensions like .ahk (AutoHotkey), .py (Python), and others commonly used to write click macros. It then analyzes the contents of those files for keywords associated with auto-clicking or input automation.
  • Active and recently run scripts: Checks for scripts that are currently running or were recently executed.
  • Gaming software folder check: Scans for Logitech G Hub, Razer Synapse, Corsair iCUE, and Glorious Core folders, and reports when they were last run.
  • DWM.exe keyword scan: Scans dwm.exe memory for the keyword "Macro." Normal results include system strings like configMacro, CRMacros, altmacro, endmacro, noaltmacro. A flag would be finding something like 198Macros, Hunt Macros, Zenith Macros, or any unrecognized/irregular macro software name in that memory space.

MeowImportsChecker

This tool scans folders like C:\ for files and folders, then analyzes their strings for patterns commonly used in external cheat and macro clients. It is NOT a mod analyzer — it operates on files outside the game, looking for cheat infrastructure present on the filesystem.

powershell -ExecutionPolicy Bypass -Command "Invoke-Expression (Invoke-RestMethod 'https://raw.githubusercontent.com/MeowTonynoh/MeowImportsChecker/main/MeowImportsChecker.ps1')"

Manual Mouse Software Check:

If MacroScanner flagged gaming software, open it manually. Navigate to profiles tied to Minecraft or javaw.exe. A flag is any macro on left-click/attack with a hardcoded repeat loop — e.g., Down → wait 10ms → Up → wait 10ms → Loop. Human clicking cannot maintain a perfect fixed-ms interval.

Phase 8: Strings Checker

Laffer's Strings Checker (referred to as "Strings") is a dedicated tool that automatically analyzes the active javaw.exe process memory for cheat-related strings. This is different from bstrings, System Informer, or Echo's String Tool — it is purpose-built to bypass the need to manually filter every string, giving you a focused output of suspicious hits directly.

What to do:

Run Strings from C:\SSTools against the Minecraft process. Review the output for anything that wouldn't naturally appear in a legitimate game client.

What is Normal (can appear, not necessarily a ban):

  • References like accurateblockplacement, wurst, meteor, or argon — these can be false flags from legitimate mods that reference other projects in metadata or class names.
  • Generic class names from Fabric/Forge APIs.
  • Standard Java library paths under Program Files or .minecraft\libraries.

What is a Flag:

  • Blatant file paths: An Argon cheat file path (e.g., C:\Users\User\argon\modules\KillAura.class) means the cheat is actively loaded in memory — not a false flag.
  • Chained cheat module names: killaura, triggerbot, autocrystal, bhop, scaffold appearing together in the same output indicates an active module list from a ghost client.
  • Ghost client keybind patterns: Strings like key.{}.toggle or key.{module}.bind do not appear in any legitimate mod. These are ghost client keybind registrations.
  • Cheat distribution URLs: Domain names pointing to known cheat sites (e.g., doomsdayclient.com) in an unexpected context.
  • Base64 blobs or encoded payloads: Indicates a loader decoding a payload at runtime.
  • Anything unnatural: If a string has no logical reason to appear in a clean Minecraft client, treat it as suspicious and cross-reference with Phase 9.
Rule of thumb: A single client name reference might be a false flag from metadata. An actual file path to a cheat, a cluster of cheat module names, or key.{}.toggle patterns are evidence.

Phase 9: System Informer & Thread Inspection

System Informer with its kernel-mode driver allows deep inspection of service threads, process memory strings, and suspended threads used by advanced bypassers to silently disable logging while services appear "Running."

Open System Informer as Administrator with the kernel driver active. For each process below: right-click → Properties → Memory → check Hide free pages and Hide reserved pages → Strings tab, set minimum length to 4, check Extended Unicode, Mapped, and Image.

DPS Process — Filter Regex

The DPS process (dps.dll inside svchost) records paths of executed files. Use these filters to find suspicious executions. Each regex serves a different detection purpose:

Filters out .exe files — shows everything except standard executables ^!![A-Z]((?!Exe).)*$
Filters out .exe and .dll files ^\\device\\harddiskvolume((?!Exe|dll).)*$
Modified extension — finds non-.exe files executed from a disk volume path ^\\device\\harddiskvolume[0-99]\\((?!exe).)*$
Localhost — shows connections or executions going through localhost ^\\device\\mup\\localhost\\*

CSRSS Process — Filter Regex

CSRSS tracks all process creation on the system. There are two CSRSS instances — use the correct one for each check:

For .exe paths — use the CSRSS with lower private bytes. Filter, then save dump to a folder and run through Paths Parser. ^[A-Z]:\\.+\.(exe)$
For .dll paths — use the CSRSS with higher private bytes. Filter, then save dump to a folder and run through Paths Parser. ^[A-Z]:\\.+\.(dll)$
Renamed Extension — finds files with non-standard extensions that were executed (modified extension bypass). Use on CSRSS with most bytes. ^[a-z]:.+\.((?!exe|pyd|manifest|dll|config|\\|cpl|microsoft-|shell).)*$

PcaSvc Process

PcaSvc (Program Compatibility Assistant) also records application execution paths. Inspect its memory strings for paths to files in non-standard locations (Downloads, Temp, hidden directories). Any .exe path referencing a folder outside standard install paths (Program Files, Windows, minecraft directories) is suspicious. Cross-reference paths found here against BAM and DPS findings.

Thread Suspension — What is a Flag:

For each service-hosting svchost, go to Properties → Threads. If any thread with a start address in the service's core DLL is Suspended, the service is silently frozen without being "disabled." These are the thread-to-DLL mappings to check:

  • SysMain → sechost.dll — Suspended = Prefetch logging frozen
  • PcaSvc → pcasvc.dll — Suspended = PCA execution recording frozen
  • DPS → dps.dll — Suspended = DPS path recording frozen
  • EventLog → wevtsvc.dll — Suspended = All event logging frozen
  • dusmsvc → wlanapi.dll — Suspended = Data usage tracking frozen
  • cdpusersvc_[numbers] → ucrtbase.dll — Suspended = CDP activity frozen

Memory Clearing Artifact:

If someone used System Informer or a similar tool to manually wipe cheat strings from memory before the SS, they leave a trace. In processes like dwm.exe, explorer.exe, or taskhostw.exe, search for the pattern ) (0x. A result like:

name.exe (4684) (0x195c66a3000 - 0x195c66c7000)

confirms that a memory region of that process was manually accessed. The process name, PID, and memory range are all recorded as a residual. This alone is strong grounds for a ban.

Phase 10: Advanced Checks

For high-confidence suspects who passed earlier phases, or when initial results are inconclusive, apply these deeper checks.

Alternate Data Streams (ADS)

ADS allows entire executables to be hidden inside innocent files (e.g., notes.txt:cheat.exe). The host file's size appears normal in File Explorer.

dir /r C:\Users\%username%\Downloads
dir /r C:\Users\%username%\Desktop

Use AlternateStreamView (Nirsoft) for a full system scan. Any stream with significant file size attached to a small host file is suspicious. ADS creation generates STREAM_CHANGE events in the USN Journal.

USB Device History

Cheats loaded from USB and never written to the main drive leave minimal on-disk traces. Run USBDeview or inspect HKLM\SYSTEM\CurrentControlSet\Enum\USBSTOR. A FAT32 or exFAT external drive connected just before the freeze is an immediate ban — those filesystems have no journaling equivalent to NTFS, making deletions from them essentially untraceable.

Cloud Storage Folders

Inspect C:\Users\%username%\OneDrive and any Google Drive/Dropbox sync folders. A cheat stored in a cloud-synced folder can be deleted remotely by an accomplice from another device while the SS is ongoing, causing the file to vanish locally with minimal immediate traces.

Kernel Live Dump (Fileless Detection)

For suspected fileless or memory-injection cheats — where no files exist on disk — create a Kernel Live Dump via System Informer and analyze it with bstrings to find loader commands, encoded payloads, and download URLs that were executed entirely in memory:

bstrings.exe -f "C:\Path\To\LiveKernelDump.DMP" --lr "(?i)(?:\b(?:powershell|cmd|wscript|cscript)\b|iwr|iex|invoke|encodedcommand|decoded|base64|github|pastebin|\.exe|\.bat|\.vbs)" -o ".\FilelessEvidence.txt"
