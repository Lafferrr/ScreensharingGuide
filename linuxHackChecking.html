Linux Hack Checking Workflow

Forensic protocols and shell commands for Linux screenshares.

Phase 1: Environment Identification

Linux is highly customizable — logs, paths, and directories vary significantly between distributions (Ubuntu, Arch, Fedora, Debian). Always identify the environment first so your commands target the correct paths.

cat /etc/os-release && uname -a
Limitations: Linux has no Prefetch, no BAM, and no USN Journal. Your primary evidence sources are shell history, process inspection, loaded libraries, filesystem timestamps, and systemd/journalctl logs. Terminal history is your equivalent of BAM — guard it carefully.

Check for Hidden Directories in Home

ls -la ~/ | grep "^\."

Look for unusual hidden directories like .meteor, .wurst, or any folder with a random-looking name. Legitimate hidden directories include .config, .local, .ssh, etc.

Phase 2: Hidden Minecraft Directories

Linux uses dotfiles to hide application data in the home directory. The -laR flag shows all hidden files, permissions, and modification times recursively.

Vanilla Minecraft

cd ~/.minecraft/mods && ls -laR

Flatpak/Snap Minecraft

If installed via Flatpak, the path is entirely different:

cd ~/.var/app/com.mojang.Minecraft/.minecraft/mods && ls -laR

Check All Common Minecraft Paths

find ~ -name "*.jar" -path "*/minecraft/*" -newer ~/.minecraft/launcher_profiles.json 2>/dev/null

Any .jar newer than the launcher profiles (set at install time) has been added or modified after installation.

Check /tmp for Cheat Artifacts

ls -laR /tmp/ 2>/dev/null | head -80

Ghost clients frequently extract payloads to /tmp before injection because it is writable by all users and typically cleared on reboot. Check for any unexpected .jar, .so, or executable files.

Phase 3: Shell History Analysis

This is the most critical phase on Linux — shell history is your primary execution record, equivalent to Prefetch and BAM on Windows.

Check All History Files

cat ~/.bash_history; cat ~/.zsh_history; cat ~/.sh_history 2>/dev/null

Check History with Timestamps

Bash can be configured to store timestamps with history entries:

HISTTIMEFORMAT="%F %T " history 2>/dev/null | tail -100

Search for Suspicious Commands

grep -i -E 'rm|wget|curl|chmod|mv|LD_PRELOAD|javaagent|inject|python|nc |netcat|base64' ~/.bash_history ~/.zsh_history 2>/dev/null

What is a Flag:

  • LD_PRELOAD: Forces a shared object (.so) to load into every process. LD_PRELOAD=/path/to/cheat.so java -jar minecraft.jar is a direct cheat injection.
  • -javaagent: Running java -javaagent:/path/cheat.jar loads an arbitrary JAR at JVM startup before anti-cheat.
  • history -c or rm ~/.bash_history: Wiping history to destroy evidence — immediate ban.
  • wget/curl to /tmp or hidden dirs: Downloading files to temporary locations shortly before the session.
  • base64 decode: A base64-decoded command executed in shell is a common fileless execution pattern.
Empty history: Both history files empty or absent on a system with active Minecraft use indicates history deletion. Treat as highly suspicious.

Phase 4: Process and Memory Inspection

Check Java Execution Arguments

ps -ef | grep java

Inspect the full command line. Look for: -javaagent: pointing to unknown JARs, -cp classpath entries from non-standard directories, or LD_PRELOAD set in the process environment.

Check Full Process Environment

Get the PID from the above command, then read the process environment for any injected variables:

cat /proc/$(pgrep -n java)/environ 2>/dev/null | tr '\0' '\n' | grep -i "preload\|agent\|inject"

Check Loaded Shared Objects (.so)

cat /proc/$(pgrep -n java)/maps | grep -i "\.so" | grep -v "/usr/\|/lib\|jdk\|jre\|minecraft"

Any .so library loaded from ~/, /tmp, or /home instead of standard system directories (/usr/lib, /lib) is a major red flag. This is the Linux equivalent of an injected DLL.

Check /proc cmdline for All Java Processes

for pid in $(pgrep java); do echo "PID $pid:"; cat /proc/$pid/cmdline 2>/dev/null | tr '\0' ' '; echo; done

Phase 5: Filesystem Integrity Checks

Recently Modified Files (Last 60 Minutes)

find ~/ -mmin -60 -type f -not -path "*/cache/*" -not -path "*/.config/pulse/*" 2>/dev/null | head -50

Check Minecraft Libraries for Tampering

find ~/.minecraft/libraries -name "*.jar" -newer ~/.minecraft/launcher_profiles.json 2>/dev/null

Search for Executable Files in Unusual Locations

find ~/ /tmp -perm /111 -type f -not -path "*/bin/*" -not -path "*/.local/share/*" 2>/dev/null | grep -v "\.sh$\|\.py$" | head -30

This finds executable files in home and temp directories, excluding standard local binaries. Any unexpected executable here — especially with a non-executable-looking name — is suspicious.

Phase 6: Systemd Journal & System Logs

Check Recent System Logs

journalctl -xe --since "1 hour ago" 2>/dev/null | tail -80

Check Specifically for Java/Minecraft Events

journalctl -xe --since "2 hours ago" 2>/dev/null | grep -i "java\|minecraft\|error\|killed\|segfault" | tail -40

Check USB Device History

dmesg | grep -i usb | tail -30

Look for USB storage devices (usb-storage, sd block devices) connected recently. An external drive plugged in just before the screenshare and then removed is suspicious, especially if no other evidence of the cheat remains.

Check for Recently Cleared Logs

ls -la /var/log/ | sort -k6,7 | tail -20

If systemd journal logs or auth logs have a very recent modification time but contain very little data, they may have been cleared.

Phase 7: Persistence & Cron Checks

Some advanced cheats install persistence mechanisms to survive reboots. Check these locations for unauthorized entries.

User Crontab

crontab -l 2>/dev/null

Systemd User Services

ls -la ~/.config/systemd/user/ 2>/dev/null
systemctl --user list-units --type=service 2>/dev/null

Any custom systemd service or cron job not associated with known software (backup tools, update managers) warrants investigation, especially if it references a Java process or a file in a suspicious path.

Shell RC Files

cat ~/.bashrc ~/.zshrc ~/.profile 2>/dev/null | grep -i "preload\|agent\|export\|alias" | grep -v "^#"

Check if LD_PRELOAD is persistently set in shell startup files — this would inject a shared object into every program the user launches, including Minecraft.

Phase 8: Mouse Software and Macros

Check for macro tools installed on the system. Linux users may use xdotool, xmacro, or AutoKey for automated clicking.

which xdotool xmacro autokey 2>/dev/null
dpkg -l | grep -i "xdotool\|macro\|autokey" 2>/dev/null
rpm -qa | grep -i "xdotool\|macro\|autokey" 2>/dev/null

If any macro tools are installed, inspect their configuration files for profiles that send rapid left-click events.
