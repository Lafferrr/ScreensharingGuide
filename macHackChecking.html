Mac Hack Checking Workflow

Forensic protocols and shell commands for macOS screenshares.

Phase 1: Environment & Limitations

Unlike Windows, macOS does not have Prefetch, BAM, or a USN Journal. The primary forensic artifacts are hidden application support directories, terminal history, system logs, and process inspection. You will rely almost entirely on terminal commands — ensure the suspect opens Terminal and expands the window fully before you begin.

Important: Confirm the macOS version and architecture first. Apple Silicon (M1/M2/M3) and Intel Macs have different library paths and dylib behavior. 
uname -m && sw_vers

Phase 2: Hidden Minecraft Directories

macOS hides the Library folder by default. Navigate via Terminal or the "Go to Folder" shortcut (Cmd + Shift + G). The -laR flag shows hidden files, ownership details, modification times, and recursively lists all subdirectories.

Vanilla Minecraft

cd ~/Library/Application\ Support/minecraft/mods && ls -laR

Look for recently modified .jar files or hidden folders starting with a dot (e.g., .meteor, .wurst). Check the modification timestamps carefully — a mod modified today on a game installed months ago is suspicious.

Lunar Client

cd ~/.lunarclient/offline/multiver && ls -la

Check All Hidden Directories in Home

List all hidden dot-folders in the home directory to find unusual client directories:

ls -la ~/ | grep "^\."

Check Minecraft Libraries for Tampering

Some cheats replace legitimate library JARs. Check modification dates on core libraries:

find ~/Library/Application\ Support/minecraft/libraries -name "*.jar" -newer ~/Library/Application\ Support/minecraft/launcher_profiles.json -type f

Any library JAR newer than the launcher profiles file (which was set at install time) has been modified after installation — immediately suspicious.

Phase 3: Terminal History Analysis

This is the most important phase on macOS. Cheaters may use terminal commands to inject dylib files, delete logs, or launch java agents. The history survives even after the cheat files are deleted.

Check ZSH / Bash History

cat ~/.zsh_history; cat ~/.bash_history

Search for Suspicious Commands Specifically

grep -i -E 'rm|wget|curl|chmod|mv|DYLD_INSERT_LIBRARIES|javaagent|dylib|inject' ~/.zsh_history ~/.bash_history 2>/dev/null

What to look for:

  • History deletion: history -c, rm ~/.zsh_history, or cat /dev/null > ~/.zsh_history — these are immediate bans.
  • DYLD_INSERT_LIBRARIES: macOS's equivalent of LD_PRELOAD. Used to force-load a dylib into a process at launch: DYLD_INSERT_LIBRARIES=/path/to/cheat.dylib java ...
  • -javaagent flag: java -javaagent:/path/to/cheat.jar -jar minecraft.jar — loads an arbitrary JAR before anti-cheat initializes.
  • Deleting files before the SS: rm -rf ~/Downloads/Vape*, rm -rf ~/.wurst, etc.
  • curl/wget to download files: Especially to /tmp or hidden directories shortly before the session.
Empty history: If both history files are empty or don't exist on a system the user claims to have used Minecraft on recently, they likely cleared history. Treat as suspicious.

Phase 4: Process and File System Activity

Active Java Processes

ps -ef | grep java

Inspect the full execution string. Look for suspicious -javaagent flags, non-standard classpath directories, or libraries loading from /tmp or user home instead of the official Minecraft libraries path.

Recent File Modifications (Last 60 Minutes)

find ~/ -mmin -60 -type f -not -path "*/Library/Caches/*" -not -path "*/Library/Logs/*" 2>/dev/null

This searches for any file modified in the last 60 minutes, excluding noisy cache and log folders. If they just deleted a cheat or moved files, metadata updates will appear here. Adjust -60 to a wider window if the session started longer ago.

Check /tmp for Cheat Artifacts

ls -laR /tmp/ 2>/dev/null | head -60

Ghost clients and loaders frequently extract payloads to /tmp before injection. Check for any .jar, .dylib, or suspicious executables here.

Check Loaded Dylibs in Java Process

Get the PID of the Minecraft process from the ps command above, then check what dynamic libraries it has loaded:

vmmap $(pgrep -n java) 2>/dev/null | grep -i "\.dylib" | grep -v "/usr/lib\|/System\|minecraft\|jdk\|jre"

Any .dylib loaded from the user's home directory, /tmp, or Downloads is a major red flag. Legitimate Minecraft libraries come from the JDK and system paths.

Phase 5: System Integrity & Security Logs

Check System Integrity Protection (SIP) Status

csrutil status

SIP should be enabled. If SIP is disabled, the user can modify protected system directories and inject code in ways that would be blocked otherwise. SIP disabled without a legitimate development reason is suspicious.

Recent System Logs

log show --last 1h --predicate 'process == "java"' 2>/dev/null | tail -50

This shows system log entries for Java processes in the last hour. Look for unusual library loads, crash reports, or security framework warnings that could indicate injection attempts.

Gatekeeper / Security Assessment

Check if the user has bypassed Gatekeeper to run unsigned executables:

spctl --status

If Gatekeeper is disabled (assessments disabled), the user can run unsigned and potentially malicious code without macOS warnings.

Check USB Device History

system_profiler SPUSBDataType 2>/dev/null | grep -A5 "Product ID\|Vendor ID\|Serial Number"

Phase 6: Mouse Software and Macros

Check the Applications folder for Logitech G Hub, Razer Synapse, or third-party macro tools like Keyboard Maestro. Inspect any profiles tied to Minecraft.

ls /Applications/ | grep -i "logitech\|razer\|corsair\|glorious\|maestro\|macro"

If macro software is found, open it and inspect profiles for any rapid-click macros bound to left-click with exact, non-human millisecond intervals.

Check for Cron Jobs / Launch Agents (Persistence)

Some cheats install persistence mechanisms to survive reboots. Check for unauthorized launch agents:

ls -la ~/Library/LaunchAgents/ 2>/dev/null
ls -la /Library/LaunchAgents/ 2>/dev/null

Legitimate LaunchAgents come from known software (Google, Adobe, Dropbox). Any .plist file here with an unfamiliar name or pointing to a script in a temp/home directory warrants investigation.
