Hack Checking Workflow

A step-by-step methodology for exposing modifications and bypasses.

Phase 1: Environment Preparation

Before executing any tools, you must prepare the suspect's environment to ensure evidence is preserved and tools run correctly.

  • Start Recording: ALWAYS record the screensharing session. Provide video proof of all findings.
  • Disable Protections: Turn off Windows Defender and all third-party Antivirus software to prevent them from deleting forensic tools.
  • Network Anonymity: If using AnyDesk, turn on a VPN to protect your IP address.

Phase 2: Initial Triage (Scripts)

Run these scripts via an Administrator PowerShell to gather quick intel on system integrity.

1. Service & Integrity Checker

Checks disabled services, event logs, prefetch tampering, and the recycle bin.

Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass && powershell Invoke-Expression (Invoke-RestMethod https://raw.githubusercontent.com/praiselily/lilith-ps/refs/heads/main/Services.ps1)

2. Mod Analyzer

Analyzes loaded JVM arguments, identifies disallowed mods, and checks for false flags.

powershell -Command "Set-ExecutionPolicy Bypass -Scope Process; Invoke-Expression (Invoke-RestMethod 'https://raw.githubusercontent.com/HadronCollision/PowershellScripts/refs/heads/main/HabibiModAnalyzer.ps1')"

Phase 3: Deep Forensic Analysis

Download the required tools using the collector script and perform manual analysis.

Tool Collector

Downloads BAMParser, Everything, JarParser, JournalTrace, MacroScanner, PrefetchParser, System Informer, and WinPrefetchView to `C:\Screenshare`.

powershell Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass && powershell Invoke-Expression (Invoke-RestMethod https://raw.githubusercontent.com/praiselily/lilith-ps/refs/heads/main/Collector.ps1)

Analysis Checklist

  1. JarParser: Analyze loaded .jar files via Prefetch and DcomLaunch strings to find unloaded mods.
  2. Journal Trace: Scan the USN Journal for `.jar`, `powershell`, `.pf`, and `.lnk` to identify deleted files, replacements, or file drops.
  3. BAM Parser / Prefetch Parser: Check for recently executed unsigned executables or tools used to clear logs.
  4. MacroScanner: Analyze mouse software (Logitech, Razer, Corsair, Glorious) for rapid click macros.
  5. System Informer (Canary): Use the kernel-mode driver. Inspect process memory strings (e.g., `csrss.exe`, `PcaSvc`) for executed paths, spoofed extensions, and loaded jars.
  6. String Analysis: Dump memory of suspected processes and run strings/YARA rules looking for cheat identifiers (e.g., killaura, triggerbot, autocrystal).
Tip: Cross-reference findings. If you find a suspicious execution in BAM, verify its deletion in Journal Trace and check if Prefetch was cleared.

Phase 4: Final Verification

If the server rules permit, use an automated verification tool like Echo or Ocean to confirm findings. Note that automated tools should supplement, not replace, manual forensics.
